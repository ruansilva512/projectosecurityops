Automação e orquestração

Instalação via Linha de Comandos

Execute os seguintes comandos no seu terminal (PowerShell ou Windows Terminal).

# 1. Clone o repositório oficial do AutoSOC+
$ git clone https://github.com/cyb3r-ops/autosoc-plus.git
$ cd autosoc-plus

# 2. Inicie o ambiente virtual com Vagrant (isto pode demorar vários minutos)
$ vagrant up
==> wazuh: Machine booted and ready!
==> suricata: Machine booted and ready!
... (outras VMs)

Provisionamento com Ansible

Após as máquinas virtuais estarem a correr, use o Ansible para instalar e configurar todo o software. Execute este comando a partir do seu ambiente WSL.

Terminal WSL (Ubuntu)
# Navegue para a pasta do projeto montada no WSL
$ cd /mnt/c/Users/SeuUtilizador/caminho/para/autosoc-plus

# Execute o playbook principal para configurar todo o ambiente
$ ansible-playbook -i inventory/hosts playbooks/deploy_all.yml
PLAY RECAP *********************************************************************
wazuh : ok=25 changed=18 unreachable=0 failed=0
suricata : ok=15 changed=9 unreachable=0 failed=0

Verificação e Acesso às Ferramentas

O seu ambiente AutoSOC+ está pronto! Verifique o estado e aceda às interfaces web das ferramentas.

# Verifique se todas as VMs estão no estado 'running'
$ vagrant status
wazuh running (virtualbox)
suricata running (virtualbox)

# Verifique a conectividade com todas as máquinas via Ansible
$ ansible all -i inventory/hosts -m ping
192.168.56.10 | SUCCESS => {"changed": false, "ping": "pong"}
192.168.56.11 | SUCCESS => {"changed": false, "ping": "pong"}
